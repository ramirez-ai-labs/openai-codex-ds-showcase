# Telemetry Schema

This file documents the columns in `telemetry_events.csv` generated by `simulate_telemetry.py`.

| Column              | Type    | Description                                                       |
|---------------------|---------|-------------------------------------------------------------------|
| session_id          | int     | Unique session (IDE session / coding session)                    |
| developer_id        | int     | Simulated developer ID (one dev can have many sessions)          |
| user_segment        | string  | Developer type: `beginner`, `intermediate`, `expert`             |
| task_id             | int     | Coding task identifier                                            |
| suggestion_id       | int     | Suggestion index within the session                              |
| timestamp           | string  | ISO8601 timestamp                                                 |
| language            | string  | Language: `python`, `javascript`, `typescript`, `go`             |
| model_version       | string  | Model label: `model_v1`, `model_v2`                              |
| latency_ms          | int     | Time from request to suggestion display (milliseconds)           |
| suggestion_length   | int     | Rough token/line length of suggestion                            |
| final_code_length   | int     | Approximate final code length after edits                         |
| accepted            | bool    | Whether the suggestion was accepted into the code                 |
| edited_after_accept | bool    | True if user edited accepted suggestion                           |
| ignored             | bool    | True if user ignored / dismissed the suggestion                   |
| compile_success     | bool    | Whether the code compiled or ran without syntax/runtime errors    |
| test_pass           | bool    | Whether associated tests passed                                   |
| error_type          | string  | `syntax_error`, `runtime_error`, `logical_bug`, or `none`         |
| hallucination_flag  | bool    | True if suggestion likely a “hallucination”/nonsense code         |

Use this dataset as the basis for productivity and UX analysis for AI coding tools.
